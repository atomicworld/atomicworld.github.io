Hexo搭建专属博客

发表于 2018-05-24 | 分类于 hexo ， hexo-theme-next |

之前在github pages上用静态html的模板写个人博客，比较复杂，要去html文件下修改样式、内容、以及很多标签等等，对于一个Java程序员实在头疼。最近之前的域名过期了，打算重新搭建博客。网上资料比较全的，感觉好用的就是 Hexo + Github Pages了，并且Hexo有很多主题可以选，不过个人建议，非前端、美工人员还是选择next模板，这个模板的支持度、热度比较高。

第一步：安装 git （提交、下载、同步代码到github神器）

不过多的介绍工具如何使用，可以参考前人的博客：

git教程：

  1. 廖雪峰老师的博客：git教程
  2. github官网：git官网
  3. git工具下载：git下载

建议先学会如何使用 git 工具，远程连接github，管理项目。

第二步：下载node.js

这里不需要会使用node.js，不是前端技术人员也不要慌，用来安装hexo

  1. nodejs下载地址：nojs官网

如果你前面两个步骤都做好了，那么你应该可以打开”命令控制台“,也就是windows的C:\windows\system32(64)\cmd.exe

  1. 输入”node -v“，可以查看安装的node.js版本
  2. 输入”git –version“，可以查看安装的git版本

第三步：安装Hexo

  1. 在电脑的一个D或E盘，新建一个文件夹，用来存放你的hexo，这个文件很重要，会生成博客的文章文件，例如：”D:\personBlog“
  2. 右键该文件夹，选择” Git Bash Here “，以文件地址为目录打开 git 工具，输入命令 ”npm install -g hexo “安装hexo使用依赖
  3. 执行成功之后，再输入” hexo init “，这样就会在 ”D:\personBlog“ 下生成hexo的文件。

查看hexo的官方文档：hexo官方文档

themes文件夹 （存放博客模板的目录，next之类的）
source文件夹 （管理hexo一些资源文件，包括tags、categories、about等，还有就是这里会生成博客的文章[_post目录]）
_config.yml文件，这个是hexo的配置
themes/next/_config.yml文件，这个是next的配置

主要弄清楚

  1. _config.yml 文件，一开始容易搞不清楚哪个文件是负责渲染哪一块，建议渲染一个效果就测试一次，备份文件，不管怎么样，第一次尝试记得一个一个来，多备份

第四步：配置next主题

之所以选next主题，就是因为配置主题这一块，有丰富的资源，国内网站比较容易找到资源。

  1. Hexo的next主题个性配置
  2. Next主题定制优化

有很多小功能、界面展示可以优化
